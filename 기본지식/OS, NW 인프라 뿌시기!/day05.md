# Day 5 - Port & Process (포트 & 프로세스)
## 📌 1. 개요

Day 4에서 서버를 여러 개 띄운다는 개념을 이해했다면,
Day 5에서는 그 “여러 개”의 정체가 무엇인지를 OS 관점에서 이해한다.

## 📌 2. 핵심 질문

> 서버를 여러 개 띄운다는 건 실제로 무엇을 의미하는가?

## 📌 3. First Principles (본질)
```
서버 여러 개 = 프로세스 여러 개 + 포트로 구분
```
## 📌 4. 프로세스 (Process)
**정의**
- 실행 중인 프로그램
- OS가 관리하는 실행 단위

**특징**
- 독립된 메모리 공간
- 서로 간섭하지 않음
- 고유 ID(PID)를 가짐

**예시**
- 브라우저 탭 하나 = 프로세스 하나
- Node 서버 하나 = 프로세스 하나
  
## 📌 5. 포트 (Port)
**정의**
같은 IP에서 요청을 구분하기 위한 번호

**핵심 개념**
```
IP = 주소
PORT = 입구(문)
```

**예시**
```
127.0.0.1:3000 → 서버 A
127.0.0.1:3001 → 서버 B
```

## 📌 6. 프로세스 + 포트 관계
```
[프로세스 A] → 포트 3000
[프로세스 B] → 포트 3001
```

**요청 흐름**
```
Client → IP:PORT → 특정 프로세스
```

## 📌 8. 실습
### 8.1 서버 코드 (Node.js)
```Javascript
const express = require("express");
const app = express();

const port = process.argv[2] || 3000;

app.get("/", (req, res) => {
  res.send(`Server running on port ${port}`);
});

app.listen(port, () => {
  console.log(`Server started on ${port}`);
});
```

### 8.2 서버 실행
```
node app.js 3000
node app.js 3001
```
### 8.3 요청 테스트
```
curl localhost:3000
curl localhost:3001
```

## 📌 9. 자주 발생하는 오류
**⚠️ 포트 충돌**
```
Error: EADDRINUSE
```
- 이미 사용 중인 포트
- 동일 포트로 여러 서버 실행 불가
**⚠️ 프로세스 개념 부족**
- 단순히 “여러 번 실행”으로 이해
- 실제로는 OS가 관리하는 독립 실행 단위
**⚠️ 포트 = 서버로 오해**
- 포트는 서버가 아니라 “접속 지점”
## 📌 10. 핵심 요약
- 서버 여러 개 = 프로세스 여러 개
- 같은 IP에서 구분하기 위해 포트 사용
- 요청은 IP + PORT 기준으로 전달됨
## 📌 11. 한 줄 정리
> 서버를 여러 개 띄운다는 것은, 여러 개의 프로세스를 실행하고 포트로 구분하는 것이다
